# Service Request Workflow

## Standard Request Flow

1. Request Submitted
User submits service request.

Examples:
- Password reset request
- Shared folder access request
- Software installation request
- Hardware/peripheral request
- New user onboarding request

---

## Request Review
Tier 1 Help Desk reviews:
- Request type
- Authorization required
- Fulfillment path
- Priority / impact

---

## Approval (If Required)
Requests needing approval routed for authorization.

Examples:
- Access requests
- Software requiring manager approval
- New resource provisioning

---

## Fulfillment
Assigned support group completes request.

Possible Fulfillment Groups:
- Tier 1 Help Desk
- Desktop Support
- Server Administration
- Network Operations

---

## Validation
Confirm:
- Request fulfilled correctly
- User confirms success
- Documentation updated if needed

---

## Closure
Request closed and recorded.

Optional:
- Knowledge article reference
- Ticket linkage to related incidents
